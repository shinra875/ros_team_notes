Этот гайд о том, как подготовить свое РОС окружение
Workspace - место (диреактория), где мы работаем с росом

> The core ROS 2 workspace is called the underlay
> "Основное рабочее пр-во роса называется подложкой/андерлей" - впервые такое слышу

локальные раб.пр-ва называются слоями

"дистро" - дистрибутив, версия роса. мы работаем с Jazzy

## 1. Сурсим начальные файлы
мы должны использовать эту команду каждый раз, когда открываем новый терминал. это нужно, чтобы получить доступ к командам рос
```bash
source /opt/ros/jazzy/setup.bash
```
вместо `.bash` нужно писать то расширение, какой терминал используете
Пример: `setup.bash`, `setup.sh`, `setup.zsh`

## 2. Добавляем строку для автоматизации
если нет желания каждый раз это писать, то можно добавить в начальный шелл скрипт эту строку
```bash
echo "source /opt/ros/jazzy/setup.bash" >> ~/.bashrc
```
чтобы это убрать (хз зачем), надо зайти в настройки шелла в системе и удалить эту строку из параметров
эту команду можно выполнить из любой директории терминала. 

## 3. Проверим переменные окружения
если есть проблемы с поиском или использованием рос2 пакетов, то нужно проверить переменные окружения
```bash
printenv | grep -i ROS
```

надо проверить эти переменные `ROS_DISTRO` и `ROS_VERSION`
```bash
ROS_VERSION=2
ROS_PYTHON_VERSION=3
ROS_DISTRO=jazzy
```

пункты 3.1 и 3.2 пока что не нужны (либо слишком экзотичны и непонятны пока что)

## Итог этого урока 
Для корректной работы ROS он должен быть корректно настроен. Это можно делать руками при каждом новом терминале или автоматизировать простенькой командой из пункта 2. 
если появляются проблемы с пакетами рос, то первое, что нужно сделать - глянуть переменные окружения и убедиться, что там все данные показаны верно
